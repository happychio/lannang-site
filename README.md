# Hosted marketing site and legal pages

These pages are generated by `scripts/build-legal.ts`. The home page is the marketing landing (what
the app is, a short screenshot demo, download links, contact, and links to the legal pages); its URL
also doubles as the store Support URL. The Privacy and Terms pages are rendered from
`legal/privacy-policy.md` and `legal/terms.md`. Both app stores require a live Privacy Policy URL,
Terms of Service URL, and a Support page. Hosting is free on GitHub Pages.

Do not edit the HTML by hand. Edit the Markdown or the screenshots, then rebuild:

```
pnpm legal:build
```

## Files

- `index.html` - marketing landing page (also the store Support URL)
- `media/` - app icon and app screenshots, copied from `assets/icon.png` and `assets/marketing/`
- `privacy/index.html` - Privacy Policy
- `terms/index.html` - Terms of Service
- `.nojekyll` - serves the HTML verbatim (no Jekyll processing)

## Updating the demo screenshots

The three phone screenshots come from `assets/marketing/` (`home.png`, `lesson.png`, `profile.png`).
Replace those files with fresh captures and run `pnpm legal:build` to copy them into `media/`. To
point the "See it in action" button at a live web demo, set `DEMO_URL` near the top of
`scripts/build-legal.ts` and rebuild.

## Go-live checklist

1. Fill every `[PLACEHOLDER]` in `legal/privacy-policy.md` and `legal/terms.md`: developer or
   company name, contact email, effective date, minimum age, and governing country / jurisdiction.
2. Have both documents reviewed by someone qualified in your jurisdiction. They are a tailored
   starting point, not legal advice.
3. Run `pnpm legal:build`. While any placeholder remains, the page is marked DRAFT and the tokens
   are highlighted, so a half-finished page is obvious. The DRAFT banner disappears once all
   placeholders are filled.
4. Merge this `docs/` folder to the default branch.
5. In the repo: Settings, Pages, Build and deployment, Deploy from a branch, pick the default branch
   and the `/docs` folder, Save.
6. After it builds, the pages are live at:
   - `https://happychio.github.io/lannang/` (support)
   - `https://happychio.github.io/lannang-site/privacy/`
   - `https://happychio.github.io/lannang-site/terms/`
7. Confirm those match `PRIVACY_URL` and `TERMS_URL` in `src/lib/legal.ts`, and use the same URLs in
   the App Store Connect and Google Play listings.

Notes:
- GitHub Pages needs a public repo, or GitHub Pro/Team for Pages on a private repo. If the repo is
  private and you do not want to change that, host the same `docs/` files anywhere static (Netlify,
  Cloudflare Pages, your own domain) and update the URLs in `src/lib/legal.ts`.
- To use a custom domain, set it in the Pages settings and update `src/lib/legal.ts` to match.
